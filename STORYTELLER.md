##ที่มาของปัญหาและความสำคัญ
<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ในยุคปัจจุบันที่เทคโนโลยี และอินเทอร์เน็ตมีบทบาทสำคัญในชีวิตประจำวัน วงการบันเทิง โดยเฉพาะอุตสาหกรรมภาพยนตร์ ได้รับความนิยมอย่างกว้างขวาง และมีการเติบโตอย่างต่อเนื่อง ภาพยนตร์กลายเป็นหนึ่งในสื่อบันเทิงที่มีอิทธิพลสูงสุดต่อผู้ชมทั่วโลก ด้วยการเข้าถึงที่ง่ายขึ้นจากแพลตฟอร์มสตรีมมิ่งต่าง ๆ เช่น Netflix, Disney+, Amazon Prime, HBO Max และแพลตฟอร์มอื่น ๆ ผู้ชมสามารถเลือกชมภาพยนตร์ได้ตามความสะดวกโดยไม่ต้องเดินทางไปโรงภาพยนตร์เหมือนในอดีต อย่างไรก็ตาม แม้ว่าจะมีภาพยนตร์จำนวนมากที่ถูกผลิตออกมาในแต่ละปี แต่ไม่ใช่ทุกเรื่องที่จะตรงกับความชอบหรือความคาดหวังของผู้ชมทุกคน
<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ด้วยเหตุนี้การรีวิวภาพยนตร์กลายเป็นส่วนสำคัญที่ช่วยให้ผู้ชมสามารถตัดสินใจเลือกชมภาพยนตร์ที่ตรงกับความสนใจของตนเอง เว็บไซต์สำหรับนักรีวิวภาพยนตร์จึงเป็นแหล่งข้อมูลสำคัญที่ช่วยให้ผู้ใช้สามารถเข้าถึงบทวิจารณ์ ข้อมูลเกี่ยวกับภาพยนตร์ และความคิดเห็นจากผู้ชมได้อย่างสะดวกสบาย โปรเจคนี้เกิดขึ้นเพื่อตอบสนองต่อความต้องการของนักรีวิวภาพยนตร์และผู้ที่สนใจติดตามบทวิจารณ์ โดยมีเป้าหมายเพื่อสร้างแพลตฟอร์มที่เปิดโอกาสให้ผู้ใช้งานสามารถแบ่งปันความคิดเห็น วิเคราะห์เนื้อหา และให้คะแนนภาพยนตร์ได้อย่างอิสระ เว็บไซต์นี้จะช่วยให้นักรีวิวสามารถเผยแพร่บทความเกี่ยวกับภาพยนตร์ รวมถึงให้ข้อมูลที่เป็นประโยชน์แก่ผู้ที่กำลังมองหาภาพยนตร์ที่เหมาะสมกับรสนิยมของตน เว็บไซต์นี้รองรับฟีเจอร์ต่าง ๆ เช่น การโพสต์รีวิวภาพยนตร์ การให้คะแนนและแสดงความคิดเห็น ระบบการจัดอันดับภาพยนตร์ยอดนิยม นอกจากนี้ยังมีการจัดหมวดหมู่ภาพยนตร์ที่ช่วยให้ผู้ใช้งานสามารถค้นหาภาพยนตร์ที่น่าสนใจได้ง่ายขึ้น ผู้ใช้งานสามารถสร้างบัญชีเพื่อเก็บบันทึกภาพยนตร์ที่เคยรีวิว แชร์บทความกับผู้อื่น และร่วมพูดคุยแลกเปลี่ยนความคิดเห็นกับชุมชนนักรีวิวภาพยนตร์ ดังนั้น โปรเจคนี้จึงเป็นการตอบสนองต่อความต้องการของผู้ที่รักการรีวิวและติดตามข่าวสารเกี่ยวกับภาพยนตร์ โดยมีเป้าหมายเพื่อสร้างแพลตฟอร์มที่สะดวก ทันสมัย และมีประสิทธิภาพในการเข้าถึงข้อมูลเกี่ยวกับภาพยนตร์ได้อย่างครบถ้วน

<br>

##จุดประสงค์ของ project 
<br>

1. เพื่อสร้างแพลตฟอร์มสำหรับการแลกเปลี่ยนความคิดเห็นเกี่ยวกับภาพยนตร์  ให้ผู้ใช้สามารถแสดงความคิดเห็น และรีวิวหนังได้อย่างอิสระ

2. เพื่อช่วยให้ผู้ใช้ค้นหาภาพยนตร์ที่เหมาะกับรสนิยมของตน โดยมีระบบจัดหมวดหมู่ ค้นหา เพื่อให้เข้าถึงเนื้อหาที่ต้องการได้ง่าย

3. เพื่อเป็นแหล่งข้อมูลสำหรับการตัดสินใจก่อนรับชมภาพยนตร์ โดยมีระบบให้คะแนนให้ดาว และรีวิวจากผู้ชมจริงเพื่อช่วยให้ผู้ใช้พิจารณาภาพยนตร์ที่น่าสนใจมากขึ้น

<br>

##ประโยชน์ที่คาดว่าจะได้รับ
<br>

1. ช่วยให้ผู้ใช้สามารถเลือกชมภาพยนตร์ที่ตรงกับความสนใจของตนด้วยระบบรีวิว การให้คะแนน และหมวดหมู่ตามประเภทของภาพยนตร์

2. ส่งเสริมการสร้างคอมมูนิตี้ของคนรักหนัง ทำให้ผู้ที่มีความสนใจคล้ายกันสามารถแลกเปลี่ยนความคิดเห็น และพูดคุยแนะนำภาพยนตร์ให้กันได้

3. เพิ่มโอกาสให้หนังอิสระหรือหนังนอกกระแสได้รับการพูดถึง เพราะผู้ใช้สามารถรีวิวหนังทุกประเภทได้อย่างอิสระ

4. ให้ข้อมูลที่เป็นกลางและหลากหลายมากขึ้น ช่วยให้ผู้ใช้ได้รับมุมมองจากผู้ชมจริง ไม่ใช่แค่จากนักวิจารณ์มืออาชีพหรือโฆษณาทางการตลาด

<br>

##ขอบเขตของโครงงาน
<br>

1. เว็บไซต์รวมรีวิวหนังสำหรับผู้ใช้ที่อยากเห็นจากผู้ใช้จริง 

2. มีระบบแสดงความคิดเห็นเกี่ยวกับหนัง และโต้ตอบกับผู้ใช้อื่นในเว็บไซต์ได้

3. มีระบบการจัดหมวดหมู่และการค้นหา

4. มีระบบการให้คะแนนติดดาว และจัดลำดับความนิยม

5. มีระบบควบคุมเนื้อหา และความเหมาะสมตามวัย

<br>

##Functional Requirement and Non-Functional Requirement
<br>

**Functional Requirement**

1. ระบบสามารถบันทึกและแสดงข้อมูลภาพยนตร์ เช่น ชื่อเรื่อง ปีที่ออกฉาย แนวภาพยนตร์ และผู้กำกับ 

2. ระบบสามารถรองรับการเพิ่ม แก้ไข และลบรีวิวภาพยนตร์โดยผู้ใช้งาน

3. ผู้ใช้งานมีฟังก์ชันการให้คะแนนภาพยนตร์ และแสดงคะแนนเฉลี่ยจากผู้ใช้งาน 

4. ระบบสามารถจัดหมวดหมู่ภาพยนตร์ตามประเภท แนวภาพยนตร์ และความนิยม 

5. ระบบรองรับการค้นหาภาพยนตร์ตามชื่อเรื่อง ผู้กำกับ หรือประเภทภาพยนตร์ 

6. ผู้ใช้สามารถแสดงความคิดเห็นและตอบกลับความคิดเห็นบนรีวิวภาพยนตร์

<br>

**Non-Functional Requirement**

1. ผู้ใช้ต้องการหน้าตาเว็บไซต์เข้าถึงง่าย 

2. ผู้ใช้อยากให้หน้าเว็บไม่เกิน 5 วิ 

3. ไม่อยากให้มีการเว็บล่มเนื่องจากการใช้งานมากเกินไป 

4. ต้องการให้ใช้เว็บไซต์ได้กับทุกระบบ 

5. หากมีการปรับปรุงเว็บไซต์อยากให้มีผลกระทบน้อยที่สุด



##Data structure
<br>

**Linked List**
- ใช้เก็บความคิดเห็นที่มีการเชื่อมโยงกัน เช่น ความคิดเห็นที่ตอบกลับกันในรีวิว 
- ใช้แสดงผลข้อมูลที่มีการเรียงลำดับตามลำดับเวลาของการโพสต์

**Queue**
- จัดการรีวิวที่มีการเพิ่มใหม่ในระบบ เช่น การแสดงรีวิวที่ถูกเพิ่มเข้ามาล่าสุดในคิว

<br>

**Trie** 
- ใช้ในการค้นหาชื่อหนังอย่างรวดเร็ว

**Matrix** 
- ใช้จัดเก็บคะแนนการรีวิวของหนังในแต่ละหมวดหมู่ เช่น คะแนนการรีวิวแยกตามประเภทต่างๆ

##อธิบายกระบวนการทำงาน โดยใช้ process, methods, and tools
<br>

**Process**
1. ศึกษาความต้องการของผู้ใช้และฟังก์ชันที่จำเป็นสำหรับเว็บไซต์

1. ศึกษาความต้องการของผู้ใช้และฟังก์ชันที่จำเป็นสำหรับเว็บไซต์

2. ออกแบบ UI/UX เพื่อกำหนดโครงสร้างและการทำงานของเว็บไซต์

3. พัฒนาเว็บไซต์ให้สามารถใช้งานได้จริง พร้อมทดสอบการทำงาน

4. จัดทำรายงานและเอกสารที่เกี่ยวข้องกับโครงการ

5. ทำ Retrospective วิเคราะห์ปัญหาและปรับปรุงการทำงานของทีม

**Method**

1. วิเคราะห์ความต้องการของผู้ใช้งานและฟีเจอร์ที่จำเป็น

2. ใช้เครื่องมือออกแบบ UI/UX

3. ใช้เครื่องมือสำหรับพัฒนา และจัดการโค้ดร่วมกัน รวมถึงการทดสอบประสิทธิภาพ

4. ทำเอกสารประกอบโครงงาน โดยใช้ Wiki on Azure DevOp เพื่อให้ทีมสามารถทำงานร่วมกันได้

5. ทำ Retrospective เพื่อติดตามความคืบหน้าและปรับปรุงกระบวนการทำงานของทีม

**Tools**

 Tools (เครื่องมือที่ใช้)

1. รวบรวมข้อมูลจากกลุ่มเป้าหมาย

- ใช้ Figma เพื่อออกแบบ UI/UX ของเว็บไซต์รีวิวหนัง

2. เขียน Code และทดสอบเว็บไซต์

- ใช้ Visual Studio Code ในการเขียนโค้ด

- ใช้ GitHub หรือ GitLab สำหรับการจัดการเวอร์ชันของโค้ด

- ใช้ Microsoft Azure สำหรับการ Deploy และทดสอบเว็บไซต์

3. ทำเอกสารประกอบโครงงาน

- ใช้ Google Drive หรือ Microsoft OneDrive ในการจัดเก็บเอกสารและแชร์ไฟล์

4. ทำ Retrospective โดยใช้ Zoom หรือ Microsoft Teams สำหรับการประชุมออนไลน์

##สรุปการประชุม Retrospective
<br> 

สรุป Retrospective ที่ 1 มีการพูดคุยดังนี้ 

ลิงค์คลิปการประชุม : https://youtu.be/J2fzecNiS1I
- **สรุปเกี่ยวกับโครงร่างของโปรเจกต์** โปรเจกต์เป็นเว็บไซต์รีวิวภาพยนตร์ โดยได้กำหนดฟีเจอร์หลัก ได้แก่ การให้คะแนนภาพยนตร์ ,การรีวิวภาพยนตร์ , การจัดหมวดหมู่ภาพยนตร์ และการค้นหาภาพยนตร์
- **ปัญหาด้านการสื่อสารภายในทีม** มีความเข้าใจที่คลาดเคลื่อนในบางประเด็น ส่งผลให้การดำเนินงานไม่เป็นไปตามที่คาดหมาย ดังนั้นจึงมีแนวทางแก้ไขโดยกำหนดช่วงเวลาสำหรับการประชุมเพิ่มเติม เพื่อให้สมาชิกสามารถอธิบายแนวคิด และทำความเข้าใจร่วมกันได้ดียิ่งขึ้น
- **ปัญหาเรื่องข้อจำกัดด้านเวลา** สมาชิกในทีมมีตารางเวลาที่แตกต่างกัน ทำให้เกิดความล่าช้าในการทำงาน ดังนั้นจึงกำหนดการนัดหมายล่วงหน้าเพื่อให้สามารถบริหารเวลาได้อย่างมีประสิทธิภาพมากยิ่งขึ้น
- **ปัญหาขาดประสบการณ์การใช้เครื่องมือ Azure DevOps** สมาชิกบางส่วนยังไม่คุ้นเคยกับแพลตฟอร์ม Azure DevOps ส่งผลให้เกิดความล่าช้าในการทำงาน

สรุป Retrospective ที่ 2 มีการพูดคุยดังนี้ 

ลิงค์คลิปการประชุม : https://youtu.be/DmisueymOwg
- **ปัญหาการทำงานโดยไม่มีแบบร่าง** เนื่องจากผู้จัดทำไม่ได้มีการร่างแบบในการทำ ui ไว้ ทำให้มีแก้ไขอยู่บ่อยความจำเป็นเวลาในการทำงานที่เพิ่มขึ้น ทำให้ส่วนประกอบต่างๆ ของ ui ไม่สอดคล้องกัน
